# OMNIA Fragility Checker\n\n[![DOI](https://zenodo.org/badge/DOI/10.5281/zenodo.20365172.svg)](https://doi.org/10.5281/zenodo.20365172)\n\nDetect unstable AI outputs across equivalent prompt variants.\n\nThis is a practical command-line tool for AI evaluation and QA automation.\n\nProblem solved:\n\n    same task + equivalent prompt variants + different outputs = fragile case\n\n## Why this exists\n\nAI systems can look correct on one prompt and fail on a semantically equivalent rephrasing.\n\nThat is not a philosophical problem. It is a concrete testing problem.\n\nThis tool reads a CSV or JSONL file containing multiple outputs for the same case and reports where the output changes when it should remain stable.\n\n## Install\n\n    pip install -e .\n\n## Run\n\n    omnia-fragility examples/sample_outputs.csv --out-dir report\n\nOutput:\n\n    report/report.json\n    report/report.csv\n    report/report.html\n\n## Use in CI\n\n    omnia-fragility examples/sample_outputs.csv --out-dir report --fail-on-fragile\n\nExit codes:\n\n    0 = no blocking fragility detected\n    2 = fragile cases detected\n    3 = critical fragile cases detected when --fail-on-critical is used\n\n## CSV input format\n\nRequired columns:\n\n    case_id,variant_id,output\n\nOptional columns:\n\n    input,expected\n\nMinimal example:\n\n    case_id,variant_id,input,output,expected\n    refund_001,base,"When will my refund arrive?","Refunds usually take 3-5 business days.","3-5 business days"\n    refund_001,rephrase_1,"How long does a refund take?","Refunds are instant.","3-5 business days"\n    refund_001,rephrase_2,"When do I get my money back?","Refunds usually take 3-5 business days.","3-5 business days"\n\n## JSONL input format\n\n    {"case_id":"policy_001","variant_id":"base","input":"Can this request be approved?","output":"Final answer: yes","expected":"yes"}\n    {"case_id":"policy_001","variant_id":"rephrase_1","input":"Is this request allowed?","output":"Final answer: no","expected":"yes"}\n\n## Classification\n\nThe tool emits case-level statuses:\n\n    STABLE\n    SURFACE_VARIANT\n    ANSWER_FRAGILE\n    NUMERIC_FRAGILE\n    CRITICAL_FRAGILE\n    INSUFFICIENT_VARIANTS\n\n## Practical use cases\n\n- LLM evaluation\n- prompt robustness testing\n- AI support-answer QA\n- model comparison\n- regression testing\n- pre-deployment checks\n- CI/CD quality gates\n\n## What this is not\n\nThis is not a general truth detector.\n\nIt does not prove whether an answer is universally correct.\n\nIt detects a narrower and concrete failure mode:\n\n    the output changes across equivalent variants when it should not\n\n## Background\n\nThis tool is derived from the OMNIA measurement ecosystem, but it is intentionally packaged as a standalone practical utility.\n\nThe user does not need to understand OMNIA to use this tool.\n\n\n\n## Citation
